# Aura 相册 App 设计与开发指导文件

## 🎨 第一部分：UI/UX 设计规范 (Modern Flat & Micro-Motion)

Aura 的设计语言摒弃厚重的液态毛玻璃，采用**现代扁平化风格 (Modern Flat)**，强调内容本身的呼吸感，结合细腻的微动效与触觉反馈，打造温润、现代、极具品质感的用户体验。

### 1. 核心视觉原则
*   **大圆角与亲和力 (Generous Radii)：** 所有的卡片、弹窗、图片缩略图、多选 Checkbox 均采用大圆角设计（如 `16px` - `24px`），消除界面的锐利感，带来舒适、现代的视觉体验。
*   **排版张力 (Typography Contrast)：** 采用大小对比极其强烈的字号与字重。例如，页面标题或日期采用 `32sp Heavy` 超大粗体，而次要信息（如文件大小、扩展名）采用 `12sp Regular` 并弱化颜色。利用排版本身作为界面的核心装饰。
*   **舒适的扁平配色 (Comfortable Palette)：** 拒绝刺眼的绝对黑白。
    *   **Light Mode:** 背景采用极其柔和的暖灰白（如 `#F7F7F9`），卡片采用纯白 `#FFFFFF`。
    *   **Dark Mode:** 背景采用深空灰（如 `#121212` 或 `#1C1C1E`），而非死黑。
    *   **点缀色 (Accent)：** 选择一种具有活力的品牌色（如 Google 蓝、灵动紫或薄荷绿），用于高亮状态、选中按钮和进度条。
*   **空间与光影 (Soft Depth)：** 虽然是扁平化，但不代表绝对扁平。对于底部的悬浮导航栏、顶部的下拉菜单、以及被长按选中的图片，使用**极其柔和、大面积的弥散阴影 (Soft Diffused Shadows)** 来体现层级关系（Z轴），不喧宾夺主，但层次分明。

### 2. 交互与微动效 (Micro-Interactions)
*   **弹性与阻尼 (Spring & Damping)：** 列表滑动到边缘、双指缩放图片到极限时，必须有舒适的弹性阻尼感。
*   **无缝视差转场 (Hero Transitions)：** 从列表点击小图进入预览页时，图片本身通过流畅的放大动画平移到预览位置，背景瞬间压暗。
*   **触觉反馈 (Haptic Touch)：** 在拖动右侧时间轴滑块、长按触发多选、双指缩放切换布局列数时，必须伴随极其精准的线性马达震动反馈。

## 🏗 第二部分：完整产品功能架构 (一字不落还原)

### 模块一：全局导航
*   **底部悬浮导航栏：** 采用胶囊状或大圆角悬浮设计，带有轻微弥散阴影。分为两个 Tab：**照片** (默认高亮) 和 **相册**。

### 模块二：图片预览页
*   **基础展示：** 默认图片适应屏幕大小展示。
*   **手势交互：**
    *   默认右滑上一张、左滑下一张。
    *   双击放大到 100% 原始图片大小，再次双击撑满屏幕。
    *   支持双指缩放、双指旋转。
    *   **边界阻尼逻辑：** 当图片被放大时，上下左右滑动只是移动图片；当滑动到当前图片的边界时，要停止移动，继续向左或向右滑动才会进入下一张或上一张。
*   **沉浸模式切换：** 单击图片进入准沉浸模式，背景变为黑色，隐藏所有 icon 按键；再次点击图片退出沉浸模式。
*   **顶部控件：**
    *   **返回按钮：** 关闭当前预览页。
    *   **元数据展示：** 显示文件名，文件名下方是图片创建时间（格式：`YYYY-MM-DD HH:mm:ss`）。时间右侧显示图片格式、标签。
    *   **沉浸式查看按钮：** 右侧 icon 按钮，点击进入沉浸模式（背景变黑，隐藏原 icon）。再次点击退出。
        *   *沉浸模式专属交互：* 进入沉浸后，底部变成两个 icon 按钮（上一张，下一张）。单击切换一张；**长按 2s 连续切换**。切换模式为直接即刻切换，无滑动或渐隐动效。
*   **底部控件：**
    *   **添加到相册：** 唤起列表弹窗，选择或新建相册并加入。
    *   **添加标签：** 唤起弹窗，选择或新建标签。
    *   **评分：** 0-5 分交互打分控件。
    *   **删除：** 点击提供选项：永久删除、删除到回收站（90天内可恢复）、移出当前相册（变更为“未分类”相册）。
    *   **更多：** 分享、设置为壁纸、详情（大小/尺寸/创建时间/修改时间/标签等）、重命名（更新文件名）、添加描述。

### 模块三：照片页 (首页 Tab)
*   **照片展示区：**
    *   **默认排序与分组：** 按添加时间排序。2/3/4/5列时按天分组显示；8/16列时按月分组显示。
    *   **时间轴定位：** 滑动页面时右侧边缘显示滑块时间轴，点击唤起时间轴选择器快速跳转，或手动拖动滑块。
    *   **无极缩放布局：** 默认瀑布流，支持双指缩放切换不同布局的图片大小。
    *   **多选模式：** 长按触发。进入后：
        *   左上角变为“退出多选”按钮。
        *   右上角变为“选择方式”按钮（按日或按月切换）。
        *   按日选择：点击日期右端全选按钮全选该天。
        *   按月选择：点击日期右端全选按钮全选该月。
        *   退出多选时取消所有选中。
    *   *多选状态下的底部 Tab 变更：* 隐藏原导航，变为：添加到相册、添加标签、评分、删除（永久/回收站/移出）、分享。
*   **顶部操作区：**
    *   **搜索：** 输入图片名称、标签搜索。
    *   **布局切换：** 四种模式：
        1.  **瀑布流 (默认)：** 等宽自动适应高度，流式排布。双指可缩放支持 2、3、4、5、8、16 列。
        2.  **自适应：** 等高自动调整宽度，填充屏幕。双指可缩放（最低 60px，最高 100% 高度），自动调整横向数量。
        3.  **宫格：** 正方形裁剪展示。双指缩放支持 2、3、4、5、8、16 列。
        4.  **列表：** 左侧缩略图，右侧文件名+扩展名、文件大小、文件尺寸、添加时间。不可缩放。
    *   **排序方式：** 支持按：添加时间、创建时间、修改时间、文件名、文件大小、文件尺寸、评分（全部支持正序/倒序）。
    *   **极客筛选 (支持多选交叉，包含/不包含)：** 图片名称、扩展名、标签、评分、文件尺寸（横向/纵向/方形/范围）、文件大小（弹窗输入范围）、添加时间（范围）、修改时间（范围）、创建时间（范围）。
    *   **更多：** 包含 最近删除（回收站）、释放空间（清理截屏/相似图片/重复内容/缓存）、设置。

### 模块四：相册页 (Tab 2)
从上到下划分为三个核心区：
*   **顶部操作按钮：** 搜索、更多（最近删除、释放空间、设置）。
*   **常用相册区 (横向 3 宫格，展示封面/名称/数量)：**
    *   **默认展示：** 全部照片、相机、截屏录屏、视频。
    *   **点击更多展开自动相册：** 标签相册、评分相册、扩展名相册、文件尺寸相册（横/纵/方形/范围）、文件大小相册（0-10, 10-50, 50+ 或范围）。
*   **自定义相册区 (横向 3 宫格)：**
    *   **默认展示：** 6 个置顶相册。
    *   **点击更多：** 显示所有用户创建的相册。
*   **回收站区：**
    *   横向撑满的列表按钮入口。
    *   进入后查看最近删除图片，右上角二次确认永久删除按钮。
    *   支持照片预览和多选。选中时底部出现两个按钮：左侧恢复，右侧永久删除。

### 模块五：设置页 (经“更多”进入)
*   **相册隐匿功能：** 开启后 App 界面伪装成音乐播放器；关闭恢复。
*   **Aura 相册图片隔离：**
    *   **开启 (沙盒模式)：** 手机保存图片默认存入 Aura，不显示在原生相册（旧原生图片在 Aura 中不可见）。
    *   **关闭：** 图片默认保存并在手机本地相册显示，不出现在 Aura。
*   **从本地相册导入：** 全屏弹窗选择导入到 Aura。导入后可勾选“是否删除本地原生相册中的照片”。自动归类至常用相册。
*   **清空缓存：** 释放应用缓存空间。
*   **一键搬家 (移出至本地)：** 将 Aura 所有照片移出到系统原生相册，Aura 内部不再保留。
*   *(补充底层逻辑)* 移入 Aura 的图片，在手机本地系统相册中绝对隐藏不显示。

## 💻 第三部分：VS Code 开发计划与技术栈建议

为了实现上述极具挑战性的大量数据过滤、复杂手势和底层隔离，技术架构必须极其强悍。

### 1. 技术栈敲定
*   **框架：** Flutter (Dart) - *完美契合多端一致的扁平化自定义 UI，以及处理多达 16 列的复杂重绘手势。*
*   **IDE：** VS Code (配合 Flutter 插件、Dart 插件、Bloc/Riverpod 代码片段生成器器)。
*   **本地数据库：** **Isar Database** 或 **ObjectBox** - *极其重要。由于你的筛选维度包含“添加时间、创建时间、修改时间、标签、评分、尺寸等”，只有这类支持复合索引、跨平台且查询速度极快的移动端 NoSQL 数据库才能胜任“包含/不包含”的毫秒级交叉筛选。*
*   **状态管理：** Riverpod + Freezed - *用于管理复杂的筛选器状态和“多选/单选/沉浸”等UI状态。*
*   **图片手势引擎：** 基于 `extended_image` 深度二次开发，它底层支持手势拦截、边界计算、双击放大，可以极大减少你在 Phase 2 预览页重写物理计算边界阻尼的工作量。

### 2. 里程碑开发计划表

#### Phase 1: 核心资产基建与“物理隔离” (P0)
*   **重点：** 先把 App 的“地基”打好，搞定图片存取的闭环。
*   利用 MethodChannel 编写原生端（Swift/Kotlin）代码。
*   实现保存图片直接写入 App 内部沙盒路径（Application Support / Internal Storage）。
*   实现“从相册导入”并调用原生 API 执行删除确认（iOS 需要用户授权 `PHPhotoLibrary` 删除权限）。
*   初始化 Isar 数据库，建立 `ImageModel`，包含 PRD 要求的所有时间戳（添加、创建、修改）、尺寸、评分字段。

#### Phase 2: “神级”预览页与极致微动效 (P1)
*   **重点：** 打造沉浸式的查看体验。
*   集成 `extended_image`，开发单图预览。
*   配置双指缩放、旋转手势。编写计算逻辑：当 `scale > 1.0` 时，拦截页面的 PageView 滑动，只有当图片的 `Rect` 触碰到屏幕边缘时，才将滑动事件交还给 PageView（实现滑动切换边界阻尼逻辑）。
*   开发沉浸模式逻辑：点击立刻隐藏上下 UI，背景通过 `AnimatedContainer` 从白/灰变为纯黑（`#000000`）。长按连续切换照片功能的定时器逻辑（Timer）。

#### Phase 3: 复杂的视图引擎与超级筛选 (P2)
*   **重点：** 照片流的展示与过滤。
*   开发照片流页面。利用 Flutter 的 `SliverGrid` 和自定义 `SliverStaggeredGrid`（瀑布流）。
*   **难点：** 双指缩放切换列数。通过外层包裹 `GestureDetector` 监听 `onScaleUpdate` 手势，动态修改 Grid 的 `crossAxisCount` (2~16)，并配合 `AnimatedSwitcher` 实现平滑过渡。
*   右侧自定义拖动滑块时间轴（监听 ScrollController 偏移量计算时间）。
*   开发强悍的交叉筛选器面板与 Isar 查询构建器。

#### Phase 4: 相册空间与批处理逻辑 (P3)
*   **重点：** 数据归类与管理。
*   常用相册的自动聚合分类视图。
*   长按多选的交互状态管理（通过 Riverpod 全局控制 selection state）。
*   按月/按日全选的逻辑判定。
*   回收站软删除机制（记录 `deletedAt` 时间戳，启动时过滤超 90 天记录进行物理销毁）。

#### Phase 5: UI 打磨与高阶设置 (P4)
*   **重点：** 让 App 看起来是顶级的。
*   全局替换为大圆角设计（`BoxDecoration(borderRadius: BorderRadius.circular(20))`）。
*   调整字号字重对比。引入轻微的阴影 (`BoxShadow` color: 黑色 withOpacity 0.05，大 blurRadius)。
*   完成“伪装音乐播放器”的路由劫持功能。
*   性能优化：缓存策略调整，列表复用优化，确保 10 万张图片滑动不掉帧。

# Aura 相册 App 工程进度与状态概览

## 1. 项目概况
*   **产品定位：** 极简扁平化、重隐私、高性能的个人相册管理 App（对标 Eagle 的管理功能）。
*   **开发环境：** Flutter (Dart) + VS Code + Android 真机调试（Min SDK: 31）。
*   **核心设计风格：** C 端审美，极简扁平，主题色：`#E70FAD`（魅影紫），大圆角，微动效。

## 2. 核心技术栈
*   **语言：** Dart
*   **框架：** Flutter (Impeller 渲染)
*   **本地数据库：** Isar NoSQL (用于高性能交叉筛选)
*   **图片加载与手势：** `extended_image`
*   **权限管理：** `permission_handler` + `photo_manager`
*   **构建：** Gradle (Kotlin DSL), `build_runner` (代码生成)

## 3. 已完成的功能与进度
1.  **环境基建：** Android SDK 配置、Kotlin DSL 语法适配、NDK 架构兼容（ABI Filters）。
2.  **底层存储：** 实现了 `IsarService` 点火引擎，定义了 `ImageModel` (图片身份证) 与 `AlbumModel` (相册身份证)。
3.  **UI 骨架：** 搭建了 `MainSkeleton`，实现了悬浮式胶囊导航栏（无文字，带同色系浅背景选中动效）。
4.  **权限与隔离逻辑：**
    *   实现了 `PhotoImportEngine`，成功打通了系统相册 -> 应用沙盒的“物理隔离”流程。
    *   编写了从系统相册读取图片并物理复制至 `getApplicationSupportDirectory()` 沙盒路径的逻辑。
    *   解决了国产安卓系统（如 MIUI）的权限拦截问题。
    *   **当前状态：** 点击主页“吸入最新照片”按钮，可成功提取本地相册图片，物理存储至 App 私有沙盒，并写入 Isar 数据库。

## 4. 关键语法与配置 (接力者注意)
*   **主题配色：** 全局统一使用 `#E70FAD` 作为 `primaryColor`，背景色 `#F6F6F8`。
*   **文件路径：**
    *   数据库模型：`lib/data/models/`
    *   导入引擎：`lib/features/photos/photo_import_engine.dart`
    *   主入口：`lib/main.dart`
*   **已知坑点：**
    *   `isar_flutter_libs` 的 `namespace` 需要在 `android/app/build.gradle.kts` 或报错路径的 `build.gradle` 中显式指定 `namespace 'dev.isar.isar_flutter_libs'`。
    *   Kotlin DSL 编译时语法较严，修改 gradle 文件需谨慎。

## 5. 接力任务 (下一阶段目标)
*   **目标 1 (瀑布流展示)：** 利用 `ImageModel` 数据库中的数据，渲染瀑布流/宫格页面，实现图片库的“极速无极缩放”浏览。
*   **目标 2 (图片筛选引擎)：** 完善基于 Isar 的多条件交叉查询逻辑（包含/不包含标签、评分、时间范围）。
*   **目标 3 (预览模式)：** 编写沉浸式预览页面，实现双指缩放、双击放大、边界触碰切换照片的物理感动效。
